USAGE of PREORDER INORDER and POSTORDER FOR BINARY TREES

The traversal strategy the programmer selects depends on the specific needs of the algorithm being designed. The goal is speed, so pick the strategy that brings the nodes required the fastest.

  1. If the need is to explore the roots before inspecting any leaves, pick pre-order because it will encounter all the roots before all of the leaves.
  2. If the need is to explore all the leaves before any nodes, select post-order because it doesn't waste any time inspecting roots in search for leaves.
  3. If the tree has an inherent sequence in the nodes, and there is a need to want to flatten the tree back into its original sequence, than an in-order traversal should be used. The tree would be flattened in the same way it was created. A pre-order or post-order traversal might not unwind the tree back into the sequence which was used to create it.

struct Node{

   Node * lptr; // Pointer to Left subtree

   int data;

   Node *rptr; // Pointer to Right subtree

};

/* Print Pre-order traversal of the tree */

void preOrder(node* r){

    if(r){

        cout << r->data << " ";    

        preOrder(r->left);

        preOrder(r->right);

    }

}

Pre-order: F, B, A, D, C, E, G, I, H

 

/* Print In-order traversal of the tree */

void inOrder(node* r){

    if(r){

        inOrder(r->left);

        cout << r->data << " ";        

        inOrder(r->right);

    }

}

In-order: A, B, C, D, E, F, G, H, I

 

/* Print Post-order traversal of the tree */

void postOrder(node* r){

    if(r){

        postOrder(r->left);

        postOrder(r->right);

        cout << r->data << " ";        

    }

}

Post-order: A, C, E, D, B, H, I, G, F

 

ALGORITHM FOR BINARY TREE CREATION FROM INORDER AND PREORDER

Let the inorder and preorder traversals be given in the arrays inorder and preorder respectively.

The function to build the tree will be denoted by buildTree(i,j,k) where i,j refer to the range of the inorder array to be looked at and k is the position in the preorder array.

Initial call will be buildTree(0,n-1,0)

The algorithm has the following steps:

  1. Traverse preorder from start. The first node is the root, then we have the left subtree and then the right subtree. Create a node with this as the element.
  2. Search the node in the inorder array. Suppose its found at x. Decrement k. k refers to the position in the preorder array we are currently at. k has to be passed by reference.
  3. Finally populate the left child and right child with the return value of the recursive calls      left child = buildTree(i,x-1,k) right child = buildTree(x+1,j,k)
  4. In the end return the node
 

Let us consider the below traversals:

Inorder sequence: D B E A F C
Preorder sequence: A B D E C F

In a Preorder sequence, leftmost element is the root of the tree. So we know ‘A’ is root for given sequences. By searching ‘A’ in Inorder sequence, we can find out all elements on left side of ‘A’ are in left subtree and elements on right are in right subtree. So we know below structure now.

                 A

               /     \

         D B E    F C

We recursively follow above steps and get the following tree.

         A

       /   \

      B     C

    /   \     /

  D    E F

     

REVERSE ENGINEERING FOR BINARY TREE CREATION USING PREORDER INORDER AND POSTORDER TRAVERSALS

 

As the usage of trees and especially binary trees increased in all the fields pertaining to data storage and connectivity, the size of these graphs increased exponentially. It is very difficult and expensive to store the data available in these trees as it is. It takes up valuable memory space as well as increases the cost of storage. A smarter and more cost productive way to store this information was needed.

The concepts of preorder, inorder and postorder traversals were developed. These alone couldn’t help in storing all the information in memory efficient ways. Owing to the growing size of the data and the difficulty in storing them efficiently, a new and optimized way was needed. If a combination of preorder, inorder and postorder traversals was available that could store the data and recreate it, with the least overhead, whenever needed, it would help in mitigating the problem as the preorder, inorder and postorder traversals would be lists, albeit long, but linear in nature and thus utilize less of the much needed memory.

 

Possible minimalist combinations for the same:-

Preorder and inorder

Inorder and postorder

Preorder and postorder

 

Possible problems faced:

 

While recreating trees using any one of these combinations it as noted that in order to get unique binary trees, inorder traversal was necessary.

The reason for this is that when using preorder and postorder traversals to create tree there was ambiguity.

Example:-

                      A             A

                     /                  \

                  B                     B

 

Preorder, and Postorder traversals are same for the trees given in above diagram.

Preorder Traversal = AB
Postorder Traversal = BA

Therefore it was necessary to have inorder traversal when creating a binary tree from its preorder, inorder and postorder traversals.

The same goes for levelorder traversals. Even the combination of preorder, postorder and levelorder could not solve the problem of ambiquity.

Level-Order Traversal = AB (for the example above)

Inorder traversal is necessary because it provides the node(root) that will divide the given traversal into the right and the left sub trees. Then the preorder or the postorder is used to determine the child nodes.
